#### 2.3 需求分析
山东财经大学是一所拥有三万余名师生的综合性财经类大学，是由山东财政学院和山东经济学院于2011年合并而来的，共有4个校区。这4个校区分布间隔距离较大，由于同学们的学业繁忙，校区间同学的交流来往成本很高，因此，“印象山财”从同学们的需求出发，为促进4个校区之间同学的交流，开发出这个App。主要包含的功能点有：  
- 寻物启事/失物招领
- 同学互助
- 校园地图
- 学校新闻

### 3. 实施计划

#### 3.1 工作任务的分解与人员分工
对于项目开发中需要完成的各项工作，从需求分析、设计、实现、测试直到维护，包括文件的编制、审批、打印、分发工作，用户培训工作，软件安装工作等，均由 Eind Studio 成员合作共同完成。

#### 3.2 进度
有关项目进度详见项目进度报表，在此不做赘述。

#### 3.3 关键问题
本项目要求开发一个符合山东财经大学同学需求的校园类手机App，要求在相对较短的时间（6月到9月份）与人员相对不充足（只有3名队员，其中1个程序员）的现状下快速完成手机App和相关服务端的开发。如何在较短的时间内开发出符合要求的项目，是我们面临的关键问题。

项目开发需要设备：Android手机、iOS设备（iPad、iPhone），MacBook 各一台。
项目开发环境：
1. Windows:   WebStorm、JDK 1.6、Android SDK、node.js
2. Mac OS X:  WebStorm、JDK 1.6、Android SDK、Xcode 5.1、node.js

#### 4.1 计算机系统支持
虽然命题没有要求打包iOS App，但是我们决定 iOS 和 Android 都打包。打包iOS需要一台Macintosh。开发所用计算机需要安装相关开发环境，对操作系统的要求一般是高于Mac OS X Mavericks 或者 Windows 7。

#### 4.2 需要用户承担的工作
本项目生成的App兼容 iOS 6.0+ 和 Android 4.0+ ，基本覆盖目前主流移动应用系统，一般情况下，用户不需要做任何额外工作即可安装此App。

### 5. 专题计划要点
1. 能够实现不同校区同学之间的交流
2. 为新生入学提供方便（地图）
3. 方便同学通过移动端查看学校通知（学校新闻）
4. 现代化、扁平化风格的用户界面（UI）
5. 简单、轻松的下载安装过程
6. 方便、贴心的其他功能（如校车时刻表等）

##### 6.2.2 实际进展与进行中的难点
由于本项目采用的是比较少见的Hybrid技术，构建App的规范、最佳时间比较匮乏，这也是作者从App framework迁移到Ionic framework的原因，具体会在*技术文档*中介绍。  
另外，Hybrid App实现图片上传也是一大难点，我们最终决定使用第三方的外链图床*贴图库* ( http://www.tietuku.com )。  

### 7. 对难点采取的对策
在开发过程中，遇到很多疑难杂症，大部分技术问题都可以通过Google、StackOverFlow、GitHub等一些技术社区可以解决，需要注意的是，国内（中文）的技术资料普遍质量较差、内容匮乏，而且本项目采用的技术作者都是外国人，因此英语在我们的开发过程中占有举足轻重的地位。
